# v1.0.36 - 请求重写规则（Rewrite Rules）

## 🎯 新功能

### #11 请求重写规则（Rewrite Rules）
持久化自动修改匹配请求的 URL/Header/Body/Status，配置一次永久生效。

**功能特性**：
- ✅ URL 重写（正则表达式匹配替换）
- ✅ 请求头修改（添加/删除/修改）
- ✅ 响应头修改（添加/删除/修改）
- ✅ 请求体修改（字符串替换或完整替换）
- ✅ 响应体修改（字符串替换或完整替换）
- ✅ 状态码覆盖
- ✅ 多个规则同时生效
- ✅ 批量操作（全部启用/禁用/清空）

**入口**：工具下拉菜单 → Rewrite Rules

**优先级**：Map Local > Auto Responder > Map Remote > **Rewrite Rules** > Breakpoint

**典型使用场景**：
1. 自动添加认证 Header（如 Authorization）
2. 自动修改请求参数（如添加 debug=true）
3. URL 重定向（如 /old-api → /new-api）
4. 响应状态码覆盖（如 403 → 200）
5. 响应 Header 修改（如添加 Cache-Control）
