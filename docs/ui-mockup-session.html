#3 会话保存与恢复

保存当前抓包会话，下次打开时恢复所有请求数据和 UI 状态

📐 技术方案

💡 核心设计：Session = 元数据 + 时间范围引用

❌ 错误方案

session 表存储 requests[]
→ 数据重复存储
→ 存储空间浪费
→ 数据一致性问题

✅ 正确方案

session 表存储时间范围
→ 零数据重复
→ 查询 requests 表获取数据
→ 数据始终一致

🗄️ 数据库设计

sessions 表结构
字段 类型 说明
id INTEGER 主键，自增
name TEXT 会话名称
start_time DATETIME 会话开始时间
end_time DATETIME 会话结束时间
request_count INTEGER 请求数量
filters_json TEXT 过滤状态 JSON
view_mode TEXT 视图模式 (list/group)
created_at DATETIME 创建时间

🔄 数据流

保存 & 加载流程

保存流程

1. 用户点击「保存当前会话」
↓
2. 弹窗输入会话名称
↓
3. 记录时间范围 + UI 状态
↓
4. 写入 sessions 表

加载流程

1. 用户点击「加载」
↓
2. 确认弹窗（提示丢失当前数据）
↓
3. 根据时间范围查询 requests 表
↓
4. 恢复请求列表 + UI 状态
