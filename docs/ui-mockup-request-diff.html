📋 设计说明

核心设计原则

1. 三色标记: 绿色(新增) / 红色(删除) / 黄色(修改)，符合行业标准
2. 并排对比: 左请求1 / 右请求2，方便逐行对照
3. 结构化 Diff: JSON 使用 jsondiffpatch 进行结构化对比，而非文本 diff
4. 概览优先: 第一个 Tab 展示差异统计，快速定位问题

交互设计

1. 选择方式: 右键菜单 / 复选框模式
2. Tab 切换: 概览 / 请求头 / 请求体 / 响应头 / 响应体
3. 导出功能: HTML 报告 / Markdown / JSON delta
4. 交换按钮: 快速切换请求1和请求2

技术实现

1. Diff 算法: jsondiffpatch (JSON) + diff (文本)
2. 状态管理: Pinia (diff-store.ts)
3. 路由: /diff?id1=xxx&id2=xxx
4. 性能优化: 大文件(>500KB)提示"过大，仅显示概览"
