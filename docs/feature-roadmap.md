# PowerCatch 功能路线图

> 抓包工具可扩展功能规划，按实施优先级分层排序
>
> **阅读说明**：从地基 → 高价值 → 中价值 → 锦上添花依次实施。同层内按编号顺序。

---

## 🏗️ 基础设施（地基，必须先做）

> 这些不是"炫酷功能"，但没有它们，其他功能的体验都会打折。

### 1. 高级过滤系统
- **说明**：在现有域名通配符过滤基础上，新增按 HTTP 方法、状态码、Content-Type、响应时间范围、请求大小、路径关键字等多维度组合过滤
- **场景**：几千条请求中快速定位目标接口；过滤出所有 4xx/5xx 错误；只看 POST 请求
- **工作量**：小
- **前置依赖**：无
- **涉及模块**：`RequestList.vue`（过滤栏 UI）+ `request-store.ts`（computed filteredRequests 扩展）
- **优先级理由**：Charles/Fiddler/Proxyman 都有，没有它书签功能的使用频率会大幅下降（有了高级过滤，书签需求降低）

### 2. 响应体智能预览
- **说明**：根据 Content-Type 自动选择预览模式——JSON 树形折叠+路径复制、图片直接渲染、Hex 查看器、HTML 渲染预览、纯文本语法高亮
- **场景**：快速查看 API 返回的 JSON 结构；确认图片资源是否正确加载；查看二进制响应
- **工作量**：中小
- **前置依赖**：无
- **涉及模块**：`RequestDetail.vue` 改造（响应体 Tab 多模式切换）

### 3. 会话保存与恢复
- **说明**：将当前抓包会话（含请求列表、过滤状态、视图模式）保存到本地数据库，并支持加载历史会话
- **场景**：分享调试场景给同事；回溯昨天的抓包记录；对比不同时间段的流量
- **工作量**：小
- **前置依赖**：无（SQLite 已就绪）
- **涉及模块**：`sqlite.ts`（会话表）+ 新增会话管理 UI + `request-store.ts`（序列化/反序列化）

### 4. 大响应体处理
- **说明**：超过阈值（默认 5MB）的响应体自动截断显示，并标记 `⚠️ 大响应体已截断`；支持"加载完整内容"按钮按需获取
- **场景**：防止前端渲染卡死；避免内存溢出；快速浏览大文件响应
- **工作量**：小
- **前置依赖**：无
- **涉及模块**：`mitm-server.ts`（响应大小判断）+ `RequestDetail.vue`（截断提示 UI）

### 5. 主题切换（明/暗模式）
- **说明**：在设置页或标题栏添加明/暗主题切换开关，持久化到 `settings-store`
- **场景**：夜间调试不刺眼；统一 IDE 深色主题
- **工作量**：小
- **前置依赖**：无（Tailwind dark mode 已配置）
- **涉及模块**：`settings-store.ts` + `TitleBar.vue` 或设置页

## 🔥 高价值（对标 Charles 核心能力）

### 6. 断点 & 请求/响应修改
- **说明**：对指定 URL 设断点，请求发出前或响应返回前暂停，允许手动修改 headers / body / status code 再放行
- **场景**：模拟后端异常、修改请求参数测试边界情况、篡改响应验证前端容错
- **工作量**：中
- **前置依赖**：无
- **涉及模块**：`mitm-server.ts`（拦截层）+ 新增断点管理 UI + Store 状态

### 7. Map Local（本地映射）
- **说明**：把某个接口的响应替换成本地文件（JSON / JS / CSS）
- **场景**：前端联调时后端接口未就绪，用本地 mock 数据先开发；修改 CSS/JS 直接生效
- **工作量**：小
- **前置依赖**：无
- **涉及模块**：`mitm-server.ts` + 新增映射规则管理 UI + Store

### 8. 请求重放 & 编辑重发
- **说明**：右键某个请求 → 复制为 cURL / 直接重发 / 修改参数后重发
- **场景**：复现偶发 bug、调试接口参数、快速验证修复
- **工作量**：小
- **前置依赖**：请求序列化（#3 会话保存做好后可复用）
- **涉及模块**：`RequestList.vue`（右键菜单）+ `RequestDetail.vue` + 新增重发逻辑

### 9. Map Remote（远程映射）
- **说明**：把 A 域名的请求转发到 B 域名（如线上 → 测试环境）
- **场景**：切换环境不用改代码、灰度测试、A/B 对比
- **工作量**：小
- **前置依赖**：无
- **涉及模块**：`mitm-server.ts` + 映射规则管理 UI

### 10. 自动响应器（Auto Responder）
- **说明**：和 Map Local 的区别——不请求真实服务器，直接用本地规则响应。支持通配符匹配、延迟模拟、自定义状态码
- **场景**：离线开发、模拟超时/500/慢响应场景、Fiddler 杀手级功能
- **工作量**：小
- **前置依赖**：Map Local 规则引擎（可复用）
- **涉及模块**：`mitm-server.ts` + 规则管理 UI

### 11. 请求重写规则（Rewrite Rules）
- **说明**：断点是一次性的，重写规则是持久的：自动修改所有匹配请求的 URL / Header / Body / Status Code
- **场景**：批量修改请求头做调试；Charles Rewrite 工具的核心功能
- **工作量**：中
- **前置依赖**：无
- **涉及模块**：`mitm-server.ts` + 规则管理 UI

## 💡 中价值（提升效率）

### 12. 请求 Diff 视图
- **说明**：选中两个请求后，提供纯文本 / JSON 结构化 diff（类似 Beyond Compare），不只是 AI 对比
- **场景**：精确对比请求头/响应体的逐行差异、JSON 结构对比
- **工作量**：中
- **前置依赖**：无
- **涉及模块**：`CompareResult.vue` 改造 + 新增 diff 算法（可引入 jsdiff 库）

### 13. 带宽限流 / 网络节流
- **说明**：模拟 3G / 4G / 弱网环境，设置延迟和丢包率
- **场景**：移动端弱网测试、验证 loading 状态和超时处理
- **工作量**：中
- **前置依赖**：无
- **涉及模块**：`mitm-server.ts`（添加延迟/丢包逻辑）+ 设置页 UI

### 14. 请求时间线（Waterfall）
- **说明**：横向瀑布图展示每个请求的 DNS / TTFB / 下载时间分布
- **场景**：一眼看出慢请求卡在哪个阶段、页面加载性能分析
- **工作量**：中
- **前置依赖**：`mitm-server.ts` 需新增阶段时间采集（需在抓包层打点）
- **涉及模块**：新增 `WaterfallView.vue` + `mitm-server.ts` 阶段时间采集

### 15. WebSocket 抓包
- **说明**：捕获和展示 WS / WSS 消息（帧级别）
- **场景**：实时通信调试、聊天/推送/游戏协议分析
- **工作量**：中
- **前置依赖**：无
- **涉及模块**：`mitm-server.ts`（WS 拦截）+ 新增 WS 消息列表 UI

### 16. DNS 覆盖
- **说明**：不改系统 hosts 文件，直接在代理层把域名指向自定义 IP
- **场景**：切换测试环境、灰度机器调试、本地验证 CDN 回源
- **工作量**：小
- **前置依赖**：无
- **涉及模块**：`mitm-server.ts` + 设置页 DNS 配置 UI

### 17. Cookie 管理器
- **说明**：按域名查看/编辑/删除 Cookie，支持导入/导出 Cookie Jar
- **场景**：调试登录态问题、复用浏览器 Cookie 做接口调试
- **工作量**：中小
- **前置依赖**：无
- **涉及模块**：新增 `CookieManager.vue` + `mitm-server.ts`（Cookie 解析）

### 18. 上游代理链（Upstream Proxy）
- **说明**：将流量转发到上级代理（公司内网代理、Charles 串联抓包）
- **场景**：企业网络环境必备；双层抓包调试 TLS 问题
- **工作量**：中小
- **前置依赖**：无
- **涉及模块**：`mitm-server.ts` + `SettingsView.vue`

## 🌟 锦上添花（差异化特色）

### 19. 单请求 AI 分析
- **说明**：单个请求出问题时点击按钮调 AI 分析错误原因（如 500 的 stack trace），不需要选两个才能用 AI
- **场景**：快速定位接口报错原因、理解复杂响应结构
- **工作量**：小
- **前置依赖**：无
- **涉及模块**：`RequestDetail.vue` + 新增单请求 AI 分析按钮

### 20. 请求统计面板
- **说明**：按域名 / 状态码 / 响应时间聚合的统计图表（饼图、柱状图、趋势线）
- **场景**：一眼看清整体健康度、错误率分布、性能热点
- **工作量**：中
- **前置依赖**：高级过滤系统（#1，否则聚合维度不够灵活）
- **涉及模块**：新增 `StatsView.vue` + 统计数据聚合逻辑

### 21. 多格式导出
- **说明**：除了 HAR，还支持导出为 cURL 脚本 / Postman Collection / JMeter 脚本
- **场景**：分享接口给后端；导入到压测工具；生成接口文档
- **工作量**：小
- **前置依赖**：请求序列化（#3 会话保存）
- **涉及模块**：新增 `export-service.ts` 扩展

### 22. HAR 导入/导出
- **说明**：兼容 Chrome DevTools 的 HAR 格式，支持导入 HAR 文件并回放请求列表
- **场景**：和其他工具互换数据、分享抓包结果给同事、离线分析
- **工作量**：小
- **前置依赖**：会话保存（#3）
- **涉及模块**：新增 HAR 解析/序列化工具 + 导入导出 UI

### 23. GraphQL 感知
- **说明**：识别 GraphQL 请求，按 operation name 分组展示，单独展示 query/mutation 内容
- **场景**：GraphQL 接口调试（不再统一显示 `/graphql`）
- **工作量**：小
- **前置依赖**：无（可与高级过滤合并为一种特殊过滤维度）
- **涉及模块**：`request-matcher.ts`（GraphQL 解析）+ `tree-builder.ts`（分组逻辑）

### 24. 请求书签 / 收藏
- **说明**：标记常用接口方便快速定位
- **场景**：从几千条请求中快速找到关注的接口
- **工作量**：小
- **前置依赖**：高级过滤系统（#1，做好后书签需求降低）
- **涉及模块**：`request-store.ts` + `RequestList.vue`（书签标记 + 过滤）
- **优先级说明**：基础设施层的高级过滤完成后，书签的使用频率会大幅下降，建议延后或砍掉

### 25. 性能监控仪表盘
- **说明**：长时间抓包时的内存/CPU 占用趋势、请求速率、数据库大小监控
- **场景**：防止 OOM；调优抓包性能；确认是否漏抓
- **工作量**：中
- **前置依赖**：无
- **涉及模块**：`main.ts`（性能指标采集）+ 新增监控 UI

### 26. 多标签页
- **说明**：同时开多个抓包会话，对比不同时间段或不同设备的流量
- **场景**：A/B 测试对比、多设备同时调试
- **工作量**：大
- **前置依赖**：Store 架构改造（多实例隔离）
- **涉及模块**：Store 架构改造 + Tab UI
- **优先级说明**：大多数场景单会话 + 高级过滤就够了，建议延后

## 📊 功能依赖关系图

```
地基（必须先做）
├── 高级过滤系统 (#1) ──→ 统计面板 (#20) 的前提
│                    └──→ 降低书签 (#24) 需求
├── 响应体智能预览 (#2)
├── 会话保存恢复 (#3) ──→ 请求重放 (#8) 可复用序列化
│                    └──→ 多格式导出 (#21) 的前提
│                    └──→ HAR 导入导出 (#22) 的前提
├── 大响应体处理 (#4)
└── 主题切换 (#5)

高价值层（可并行）
├── Map Local (#7) ──→ 自动响应器 (#10) 可复用规则引擎
├── 断点修改 (#6)
└── 请求重写 (#11)

中价值层
├── Waterfall (#14) ──→ 需要 mitm-server 新增阶段时间采集
└── 统计面板 (#20) ──→ 需要高级过滤 (#1)
```

## 🛠️ 非功能性需求（不绑定版本，持续迭代）

| 项目 | 说明 | 触发条件 |
|------|------|---------|
| 搜索性能优化 | 万级请求下的索引，virtual-scroller 调优 | 单会话 >5000 条请求 |
| 内存管理 | 长时间录制不泄漏，LRU 淘汰策略 | 录制 >30 分钟 |

| SQLite 性能 | WAL 模式、索引优化、分页查询 | 会话 >10 万条 |
| 证书管理 UI | 一键安装/信任/导出 CA 证书 | 新用户首次配置 |
