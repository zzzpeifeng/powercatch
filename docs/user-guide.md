# 用户使用指南

## 1. 简介

PowerCatch 是一款基于 Electron 的 HTTP/HTTPS 抓包调试工具，支持流量拦截、查看、AI 智能分析。

---

## 2. 快速开始

### 2.1 安装应用

**方式一：下载安装包（推荐）**

1. 前往 [Releases](https://github.com/zzzpeifeng/powercatch/releases) 页面
2. 下载对应平台的安装包
3. macOS 用户：下载 `.dmg` 文件，打开后拖拽到「应用程序」文件夹

**方式二：从源码构建**

```bash
# 克隆仓库
git clone https://github.com/zzzpeifeng/powercatch.git
cd powercatch

# 安装依赖
npm install

# 开发模式（热更新）
npm run electron:dev

# 构建安装包
npm run electron:build
```

### 2.2 配置代理

1. 打开 PowerCatch 应用
2. 点击左下角「设置」图标
3. 配置代理端口（默认 `8888`）
4. 点击「启动代理」按钮

### 2.3 安装 HTTPS 证书

为了解密 HTTPS 流量，需要安装根证书：

1. 在设置页面点击「生成证书」按钮
2. 按提示安装证书到系统信任区
3. macOS 用户：在「钥匙串访问」中信任该证书

### 2.4 配置手机抓包

1. 确保手机和电脑在同一网络
2. 在手机 WiFi 设置中配置代理：
   - 服务器：电脑的 IP 地址
   - 端口：代理端口（默认 `8888`）
3. 打开手机浏览器，访问任意 HTTPS 网站，确认流量被捕获

## 3. 主要功能

### 3.1 HTTP/HTTPS 抓包

**功能说明**：
- 自动拦截所有 HTTP/HTTPS 请求和响应
- 支持全量流量捕获
- 实时显示抓包记录

**使用步骤**：
1. 启动代理
2. 在浏览器或手机上访问任意网站
3. 在 PowerCatch 主界面查看抓包记录

**过滤功能**：
- 按域名过滤：在搜索框输入域名关键字
- 按请求方法过滤：点击方法标签（GET/POST/PUT/DELETE）
- 按状态码过滤：点击状态码标签（2xx/3xx/4xx/5xx）

### 3.2 请求详情查看

**功能说明**：
- 分 Tab 展示请求信息
- 支持 JSON 高亮显示
- 支持响应预览

**查看步骤**：
1. 在抓包记录列表中点击任意请求
2. 在右侧面板查看详细信息：
   - **请求头**：HTTP 请求头
   - **请求体**：请求参数（JSON/Form/RAW）
   - **响应头**：HTTP 响应头
   - **响应体**：响应内容（自动格式化）

**JSON 高亮**：
- 自动识别 JSON 格式
- 支持折叠/展开
- 支持复制为格式化 JSON

### 3.3 🤖 AI 智能分析（**重构后 - 纯 AI Agent 模式**）

#### 3.3.1 功能概述

**AI 分析功能**可以自动分析 API 请求对应的后端代码实现，帮助你：
- 快速定位 Handler 函数
- 理解业务逻辑（参数校验、权限校验、数据库操作等）
- 生成测试命令（curl）和断言代码（Python）

**重构亮点**（v1.1.0）：
- ✅ **更精准**：AI 自主搜索代码，不再依赖正则表达式
- ✅ **更流畅**：流式输出分析过程，实时查看进度
- ✅ **更稳定**：移除 ScanWorker，不再出现卡死问题

#### 3.3.2 配置 AI 模型

**前置条件**：
- 需要 OpenAI 兼容的 API Key（支持 DeepSeek、通义千问等）

**配置步骤**：
1. 打开设置页面
2. 找到「AI 模型配置」区域
3. 填写以下信息：
   - **API 地址**：`https://api.deepseek.com`（以 DeepSeek 为例）
   - **API Key**：你的 API Key
   - **模型名称**：`deepseek-chat`
4. 点击「保存」按钮

#### 3.3.3 使用 AI 分析

**分析步骤**：

1. **选择代码仓库**
   - 在主界面点击「选择仓库」按钮
   - 选择本地代码仓库的根目录
   - 应用会自动读取仓库结构

2. **输入 API 请求信息**
   - **请求方法**：GET / POST / PUT / DELETE
   - **请求路径**：`/api/v1/users/:id`

3. **开始分析**
   - 点击「AI 分析」按钮
   - 观察**实时分析过程**（新增功能！）：
     - 看到 AI 的"思考过程"
     - 看到 AI 调用的工具（list_directory、read_file 等）
     - 看到工具执行的结果

4. **查看分析结果**
   - **分析报告**：Markdown 格式的详细分析
   - **调用链路**：从路由注册到 Handler 的完整调用链
   - **场景分析**：正常流程、参数校验失败、权限校验失败等
   - **curl 命令**：可直接运行的测试命令
   - **Python 断言**：可用于自动化测试的断言代码

#### 3.3.4 AI 分析过程实时显示（**新功能**）

**功能说明**：
- 在分析过程中，实时显示 AI 的"思考过程"
- 显示 AI 调用的工具和参数
- 显示工具执行的结果

**界面说明**：
```

│  > 正在查看项目结构...            │
│  > 找到路由定义文件...            │
│  > 正在读取 Handler 代码...      │

│  3. read_file("router.go")        │
│     ✅ 成功                        │
└─────────────────────────────────────┘
```

#### 3.3.5 支持的分析场景

**场景 1：正常流程（200 OK）**
- 描述：参数正确、权限通过、数据库查询成功
- 输出：curl 命令 + Python 断言

**场景 2：参数校验失败（400 Bad Request）**
- 描述：缺少必填参数、参数格式错误
- 输出：curl 命令（带错误参数）+ Python 断言

**场景 3：权限校验失败（401 Unauthorized）**
- 描述：未登录、Token 过期、权限不足
- 输出：curl 命令（无认证信息）+ Python 断言

#### 3.3.6 注意事项

**分析深度**：
- 当前只做**深度模式**（全面分析）
- 分析时间取决于代码仓库大小（通常 10-60 秒）
- **不限时**：不会因为分析时间长而超时

**API 限流处理**：
- 如果遇到 API 限流（429 错误），会显示友好提示
- 建议：等待 1 分钟后重试，或切换到其他 API Key

**代码仓库要求**：
- 必须是完整的本地代码仓库（已克隆到本地）
- 支持 Go 语言项目（后续版本会支持更多语言）

## 5. 常见问题

### 5.1 代理无法启动

**可能原因**：
- 端口被占用
- 权限不足（macOS 需要管理员权限）

**解决方案**：
1. 更换代理端口（如改为 `9999`）
2. macOS 用户：右键应用 →「显示包内容」→ 以管理员权限运行

### 5.2 HTTPS 流量无法解密

**可能原因**：
- 证书未安装
- 证书未信任

**解决方案**：
1. 重新生成证书
2. macOS 用户：在「钥匙串访问」中找到该证书 → 右键 →「显示简介」→ 设置为「始终信任」

### 5.3 AI 分析失败

**可能原因**：
- API Key 错误
- API 地址错误
- 网络连接有问题
- API 余额不足

**解决方案**：
1. 检查 API 配置（设置页面）
2. 点击「测试连接」按钮
3. 查看错误提示信息
4. 确认 API 余额充足

### 5.4 AI 分析过程卡住

**可能原因**：
- 代码仓库过大
- API 响应慢
- 网络连接不稳定

**解决方案**：
1. 等待 2-3 分钟（分析大型项目需要时间）
2. 如果超过 5 分钟无响应，刷新页面重试
3. 检查网络连接
