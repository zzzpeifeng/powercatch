## 抓包500后停止增长 Bug 修复

**Bug 描述**：用户报告抓包已捕获数到达 500 后不再增加，无法继续抓包。

**根本原因**：
- `RequestList.vue` 没有虚拟滚动，500+ DOM 节点全量重渲染占满浏览器主线程
- IPC 事件回调被阻塞，UI 假死，看起来像停止抓包

**修复方案**（BugFix 快捷路径：TeamCreate → Engineer → QA → 关闭）：
1. **P0 虚拟滚动**：`RecycleScroller` 替换 `v-for`，DOM 节点始终 ~15-25 个
2. **P1 Store 性能优化**：
   - Map 索引 O(1) 查找替代 findIndex
   - push 替换 unshift（O(1) 摊销）
   - 淘汰逻辑优化，避免大数组拷贝
3. **P2 动态 flush 间隔**：< 1000 条 50ms，1000-3000 条 200ms，> 3000 条 500ms
4. **附加 P2 Bug 修复**：QA 验证发现 `selectRequest()` 未设置 `request.selected = true`，导致超 5000 条时选中请求被误淘汰，已修复

**修改文件**：
- `src/main.ts` — 新增 vue-virtual-scroller CSS 导入
- `src/components/RequestList.vue` — RecycleScroller 重写
- `src/stores/request-store.ts` — 全面重写（Map 索引 + push + 动态 flush + selectRequest 修复）
- `src/styles/main.css` — 虚拟滚动样式
- `src/stores/__tests__/request-store.test.ts` — 60 个测试用例

**验证结果**：
- TypeScript 类型检查 ✅ 通过
- `npm run build` ✅ 通过
- 测试 60/60 全部通过
- 团队 `software-bugfix-capture-limit` 已关闭清理

**技术栈**：Electron + Vue 3 + TypeScript + Pinia + vue-virtual-scroller
